解析面板

請點擊左側 Prompt 的內容
以查看詳細的結構分析。

1

角色定義與動態注入

這是 Prompt 的開場 (System Message)。使用 變數插值 (Placeholders) 技術（如 {{agent_name}}），讓同一份 Prompt 能適應不同場景。

目的： 確立 AI 身份並建立對話語境 (Context)。
2

行為準則 (Rules) - 核心邏輯

這是 Prompt 最關鍵的部分。它規範了 AI 的「思考方式」。重點包含：

  • 防幻覺： 只能使用定義好的工具，不能瞎掰。
  • 思維鏈 (CoT)： 要求先規劃 (Plan) 再行動。
  • 用戶體驗： 強制使用人類可讀名稱。
目的： 確保 AI 的行為可控、安全且符合業務邏輯。
3

少樣本學習 (Few-Shot Example)

提供一個完整的 JSON 資料結構範例。這比用文字描述規則更有效，能讓 AI 直接「模仿」正確的輸出格式。

目的： 解決 Schema Alignment (資料結構對齊) 問題，降低解析錯誤。
4

結構化輸出規範

採用 前後端分離 的思維。AI 輸出的不是直接給用戶看的文字，而是一個包含 message (前端顯示用) 和 summary (系統紀錄用) 的 JSON 物件。

目的： 便於前端程式解析 AI 回應，並構建豐富的 UI 介面。
5

工具調用與意圖映射

定義「自然語言意圖」與「後端函數」的對應關係。

  • save_activity：對話結束後的存檔點。
  • test_email：具體的測試觸發器。
目的： 將抽象的用戶請求轉化為具體的程式執行入口。
6

工具定義與澄清模板

這是 防禦性編程 的一部分。提供一個標準的工具定義模板 (Name, Function, Params)。

如果系統傳入的工具定義不清楚，AI 懂得「報錯」或「要求澄清」，而不是瞎猜參數類型。

目的： 提高系統的魯棒性 (Robustness)，避免執行錯誤的函數調用。
7

上下文與數據注入 (Context & State)

這是 Prompt 的動態數據區。透過插入變數，讓 Prompt 具有 狀態意識 (State Awareness)。

  • Context_Data：用戶歷史偏好。
  • Activity_frame：當前正在編輯的 JSON 狀態。
目的： AI 不是從零開始，而是基於當前的資料狀態進行增量修改。
